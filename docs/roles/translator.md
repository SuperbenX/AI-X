# 翻译 / 本地化（Translator / Localization Specialist）

> 用 AI 把「准确 + 优雅 + 在地化」落到每一次交付  
> （SEO 简介 180–220 字）  
> 翻译与本地化不仅是“把语言换成另一种语言”，而是把**语义、风格、文化与场景**一起搬过去：术语一致、语气匹配、法规合规、品牌统一。本模块从 **需求→术语→机器翻译→人工修订→QA→交付** 的全链路出发，提供可复制的 Prompt 卡片、工具速查表与流程 SOP，帮助你在短时间内达到专业水准，兼顾**效率、质量与风险控制**。

## 🎯 核心目标
- 明确翻译 / 本地化关键环节：**需求澄清 → 术语与风格 → MT 生成 → 人工修订 → QA → 交付**  
- 展示 AI 在**术语抽取、风格迁移、批量翻译、字幕/多媒介本地化**中的应用  
- 提供**流程化方法 + Prompt 卡片**，降低试错成本，提升可复用性  
- 强调**合规（隐私/版权）与品牌一致性**，避免“直译感”和事实幻觉

## 🧭 爆款输出流程（卡片式）

**Step 1 — 需求澄清（Scope）**  
- **要点**：目标语言/地区、读者画像、语气（正式/友好/学术）、交付格式（Doc/Markdown/SRT）、截止时间  
- **工具**：ChatGPT / Claude、Notion  
- **产出**：需求确认单 + 清晰的交付规范

**Step 2 — 术语与风格（Term & Style）**  
- **要点**：抽取关键词/术语表、建立品牌用语与禁用词、生成风格指南  
- **工具**：ChatGPT、Zotero/Notion、DeepL Write  
- **产出**：术语表（双语）+ 风格卡

**Step 3 — 机器翻译 + Prompt（MT Draft）**  
- **要点**：分段/分章节、指定领域（法律/医疗/技术）、控制长度与格式  
- **工具**：ChatGPT / DeepL / Claude  
- **产出**：初稿

**Step 4 — 人工修订（PEMT）**  
- **要点**：术语一致性、语序自然、品牌统一  
- **工具**：Grammarly / DeepL Write、Claude  
- **产出**：修订稿

**Step 5 — 质量检查（QA）**  
- **要点**：拼写/标点、数字/单位、专名/链接、事实核验  
- **工具**：LanguageTool、ChatGPT  
- **产出**：QA 报告

**Step 6 — 交付与归档（Delivery）**  
- **要点**：交付清单（主文档/图像内文字/字幕）、版本号、可追溯记录  
- **工具**：Notion、Git、Canva/Photoshop  
- **产出**：最终包

## 🧩 高频 Prompt 模板（进阶版）

### 📌 技术文档 → 中英对照翻译
你是资深技术译者。任务：将以下技术文档从{源语言}译为{目标语言}，保留 Markdown/列表/表格结构。  
条件：沿用术语表，单位/数字不要改动，语气专业简洁。  
输出：中英对照 + 新增术语表 + 修订说明。

---

### 📌 品牌文案本地化
你是品牌本地化专家。任务：将{品牌/产品}文案本地化到{目标市场}。  
条件：遵循品牌风格卡，避免直译，适配当地表达与合规。  
输出：直观口语版 / 正式文案版 + 本地化注意点。

---

### 📌 术语表自动抽取
你是术语管理员。任务：从以下文本抽取术语并给出标准译名。  
输出表格：原文术语 | 标准译名 | 说明 | 备注。

---

### 📌 字幕 SRT 生成
你是字幕本地化专家。任务：将以下字幕从{源语言}译为{目标语言}。  
条件：保留时间轴，句长≤{N}字。  
输出：合法 SRT 文件。

---

### 📌 QA 一致性检查
你是翻译 QA。任务：对“原文-译文”进行一致性检查。  
检查项：术语一致性、数字/单位、专名/链接、事实指涉。  
输出：问题清单（序号/问题/建议）。

---

### 📌 风格迁移（学术/友好/法律）
你是文体编辑。任务：将以下内容重写成{目标风格}，保持事实与术语不变。  
输出：改写稿 + 变化说明。

---

### 📌 合规敏感性审查
你是合规审读员。任务：从{行业}合规角度审查译文的风险点。  
输出：风险点清单 + 修改建议。

## 🧪 实用案例
- **技术白皮书**：ChatGPT 抽取术语 → DeepL 初稿 → Grammarly 修订 → ChatGPT QA，**交付周期缩短 40%**。  
- **跨境品牌官网**：品牌卡建立 → 本地化（UK/US/JP）→ 图片内文字替换 → 法务审读，**跳出率下降 18%**。  
- **字幕本地化**：Aegisub 时间轴 → AI 译制 → 口语化重写 → 二审，**用户好评率提升**。  
- **应用内文案**：提取 i18n 字符串 → 翻译 → TM 更新 → 批量回写，**协作成本下降**。

## 🔑 进阶玩法
- **项目级术语库**：每次交付自动更新，建立团队记忆。  
- **多地区版本**：同一语种做地区化（zh-CN/zh-TW、en-US/en-GB）。  
- **自动 QA 流程**：提交即触发术语/数值/格式检查。  
- **检索增强**：Perplexity 联动事实核验。  
- **图片/视频文字本地化**：统一字体、断行规范、字号/行高。  

---

## ⚠️ 常见坑 & 应急方案
- **直译痕迹** → 增加“读者画像 + 文体”约束。  
- **术语混乱** → 前置“术语抽取 + 标准译名”。  
- **事实幻觉** → 数据/引用需核验，标注来源。  
- **合规风险** → 特殊行业必须人工复核。  
- **格式丢失** → 指令中要求保留 Markdown/表格/列表。
